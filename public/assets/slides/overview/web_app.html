Overview of this web app

  • From top to bottom, the app is divided into three sections - a utility toolbar, an editor, and a set of component buttons.
  • The utilities on left of the toolbar are divided into four sections on the left - user, clients, issues, and sessions.
    The three buttons on the right are to adjust the heights of the editor and component buttons sections, and load the instructions.
  • Most people will not concern themselves with the client utility, and the only client they will ever work on is "Self."
  • A user can work with multiple clients, each client can have multiple issues, and each issue is worked on in multiple sessions.
  • The client, issue, and session sections each have an entry selection menu.
    By default, the client selection menu button shows Self, and the issue selection menu button shows "New Issue." Any new client will have a default name of "New Client."
    The selection menu button for the sessions section shows the time the selected session was created.
  • The session menu shows all the sessions for the currently selected issue, so when a different issue is selected, the entries update in the session selection menu.
  • The rename button () renames the currently selected entry in that section, and the new button () will create a new entry in that section.
  • The editor is where the reasons for the currently selected session are shown and entered.
    You can move the cursor with the arrow keys on the keyboard, and typing any letters or numbers will add those characters at the cursor location.
  • The component buttons section is divided into button groups on the left, and the list of component buttons on the right.
    Clicking a button group will scroll the component button list to that section.
    Clicking a component button will add that component at the cursor location in the editor.
