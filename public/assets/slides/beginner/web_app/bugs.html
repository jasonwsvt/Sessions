Current bugs I'm aware of

This page is updated when a bug is discovered or fixed.

  • The new user button should disable itself when there is a user with "newuser" as the username, but this isn't always working properly.
  • Sometimes the component buttons scrollbar shows on top of the info utility window.
  • In the instructions, if the slide goes below the bottom of the page, it scrolls the whole page, not just the slide.
  • In the data manager, the records table needs to be cleaned up, like when a session has longer lines.
  • In the data manager, deletion, merging, and importing all need to be tested and bug-checked further.
  • In the data manager, there is currently no validation for loaded data, so invalid data will cause an error.

Ideas for future app features

  • Integrate the info utility with the component buttons and automatically show only pertinent groups when showing particular slides.
  • Add the option for server-side data storage and user accounts so users don't need to be limited to one browser on one computer.
