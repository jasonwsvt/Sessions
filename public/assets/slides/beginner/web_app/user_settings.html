Web App - User Settings

The User Settings menu is accessible by clicking the top left button on the screen.

The text on the button is the username of the logged in user, and this defaults to newuser.

The fields in this menu are username, current password, new password, verified new password, and email address. The current password field isn't visible unless it has a value and hasn't yet been verified.

There are also drop-down lists for local and server storage frequencies, as well as checkboxes for making the account hidden in the login menu, and automatically logging in (which is only useful if there are multiple users).

The text in the bottom bar provides advice for fixing errors.

Buttons appear when entry combinations are appropriate for specific actions.

When initially writing the app, I hoped to have a server-side database and allow users to have server-side accounts and store data that could be accessed by logging in from any computer.

I then decided it was more important to get the app as quickly as possible into a functional state where users could learn the method and practice it, and later on, the app could then be improved based on the interests of the users.

In any case, at this point there's no real need to modify an app account unless there are multiple people using the same computer account.

Alternatively, I'd recommend using a different browser or creating an individual computer account you could use, as it would be more private and secure.

One nice feature here is that you can change the local storage frequency, or disable it.

It's important to note that if you disable the storage frequency, all the data is removed from storage, so if you want to save it you'll need to either set the storage frequency again or export it from the data manager. Otherwise it'll be gone when you close the app.
