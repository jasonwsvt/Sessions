Web App - Client, Issue, and Session Utilities

The utilities for clients, issues, and sessions are basically the same.

I guess the only concept you need to know is that there's a one-to-many relationship from users to clients, clients to issues, and issues to sessions.

Most people will not use any other client than Self, at least until they become more of an advanced user.

The Picker Menu

The sections for Clients, Issues, and Sessions all have a picker menu.

For example, the client picker shows all the clients for the current user.

Similarly, the session picker shows all the sessions for the current issue.

There are options to sort and search in each of these menus.

The Rename Menu

You can rename the currently selected client or issue using the rename button in that session.

The sessions section doesn't have a rename button because the name is automatically generated from the time it was created.

The New Button

The new button will create a new client, issue, or session.

The default name for clients is New Client, and the default name for issues is New Issue.

If there is a client or issue with the default name, the new button for that section is disabled. To re-enable it, rename the issue.
